2020-07-09|Bug
在启动项目时，我们常常会碰到这个错误： Error creating bean with name 'xxxx': Unsatisfied dependency 多数情况下是因为包名搞错了，如何排查？ 检查@Controller、@Service、@Autowired等注解是否使用正确； 配置文件中的包是否扫描； 根据name后面的名字，找到对应的文件，是否.xml文件中每一个包名都写对了？ 这里分享一个IDEA的小技巧： Ctrl + Shift + F全局搜索，比如我们这个文件是从别人那复制过来的，别人的包名肯定有和我们自己的包名不一样的部分，那么我们可以全局搜索这一部分，找到后再替换正确的包名。 我们在复制别人的代码时，常会出现这种错误，最常见的就是包名没有写好，可能是忘记改了某处的包名导致了这个错误。
