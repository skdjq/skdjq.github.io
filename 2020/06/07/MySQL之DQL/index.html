DQL—-数据查询语言，查询不会修改数据库表记录！

本文所有内容均不会修改数据库表记录！

基本查询

字段（列）控制

  • 查询所有列：select * from 表名;

    其中*表示查询所有列。

  • 查询指定列：select 列1（,列2,...,列n）from 表名;

  • 完全重复的记录只一次：select distinct * 列1（,列2,...,列N）from 表名;

  • 列运算:

    • 加减乘除：select (*) 列名 运算 from 表名;

      例如：select sal*1.5 from emp;

      带上*是显示所有的列，如果不带*则只显示修改的列。

    • 字符串类型的连续运算：select concat(列名1，列名2,...) from 表名;

起别名： 列名后加as加别名（as可以省略）

如：name (as) 姓名

条件控制

条件查询

与前面介绍的update和delete语句一样，select语句也可以使用where子句来控制记录。

例如：select name,sal,comm from emp where sal > 10000 and comm is not null;

模糊查询

关键字：like

当你想查询姓张，并且姓名一共两个字的员工时，这时就可以使用模糊查询：select * from emp where name like '张_';

当你想查询姓张的所有员工，无论名字几个字时，这时就可以：select * from emp where name like '张%';

注意：一个下划线_只能代表一个字符，而一个%却能代表从0至n个字符。

排序

关键字：order by

升序：asc

降序：desc

升序
select * from where emp order by sal asc;

  • 按sal排序，升序！
  • 其中asc是可以省略的

降序
select * from where emp order by comm desc;

  • 按comm排序，降序！
  • 其中desc不能省略

使用多列作为排序条件
select from where emp order by sal asc, comm desc;

  • 使用sal升序排，如果sal相同时，使用comm的降序排

聚合函数

聚合函数用来做某列的纵向运算。

1.count
select count (*) from emp;
–> 计算emp表中所有列都不为NULL的记录的行数
select count (comm) from emp;
–> 计算emp表中comm列不为NULL的记录的行数

2.max
select max (sal) from emp;
–> 查询最高工资
3.min
select min (sal) from emp;
–> 查询最低工资
4.sum
select sum (sal) from emp;
–> 查询工资合
5.avg
select avg (sal) from emp;
–> 查询平均工资

分组查询

分组查询是把记录使用某一列进行分组，然后查询组信息。

关键字：group by

例如：查看所有部门的记录数。

select deptno, count() from emp *group by** deptno;
–> 使用deptno分组，查询部门编号和每个部门的记录数

select job, max(sal) from emp group by job;
–> 使用job分组，查询每种工作的最高工资

组条件:

关键字：having

以部门分组，查询每组记录数。条件为记录数大于3
select deptno,count() from emp group by deptno having count() > 3;

关键字出现的顺序：

select from where group by having order by

limit子句(方言)

limit用来限定查询结果的起始行，以及总行数。

例如：查询起始行为第5行，一共查询3行记录
select * from emp limit 4, 3;
–> 其中4表示从第5行开始，其中3表示一共查询3行。即第5、6、7行记录。

  • 查询第N页
  • 每页记录数：M行

公式：limit (当前页-1) * 每页记录数，每页记录数

如：

查询第三页，每页十行：limit (3-1) * 10,10

查询第17页，每页8行：limit (17-1) * 8, 8
