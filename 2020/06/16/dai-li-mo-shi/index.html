静态代理

静态代理：由程序员创建或特定工具自动生成源代码，也就是在编译时就已经将接口，被代理类，代理类等确定下来。在程序运行之前，代理类的.class文件就已经生成。

为了更好的帮助理解静态代理，我这里举一个小小的例子：

张三卖房不会挨家挨户的问谁需要房子，也不会和每个要买的人都重复的介绍房子，那么张三会把卖房的任务交给中介（代理），由中介（代理）和各买家交流，至此，中介（代理）的目的和张三的目的都是一样的，那就是卖房。在这里，中介就是张三的代理。

首先我们创建一个公共的Person接口，该接口被中介和张三共同实现：

public interface Person {
    void sellHouse();
}

紧接着我们再创建Host（房东）和HostProxy（中介）的实体类：

Host类：

public class Host implements Person{
    private String name;
    public Host(){

    }
    public Host(String name){
        this.name = name;
    }
    public void sellHouse() {
        System.out.println(name + "卖房得50w");
    }
}

HostProxy类，这个类也实现了Person接口，但是还另外持有一个Host类对象，由于实现了Peson接口，同时持有一个Host对象，那么他可以代理Host对象执行卖房（执行sellHouse()方法）的行为。

public class HostProxy implements Person{
    private Host host;
    public HostProxy(){
    }
    public HostProxy(Host host){
        this.host = host;
    }
    public void sellHouse() {
        System.out.println("我是中介");
        host.sellHouse();
        System.out.println("您需要支付我一些费用");
    }
}

如上面的代码，在代理类里面，我们还可以加上一些别的东西。

测试：

public class MyTest {
    public static void main(String[] args) {
        Host host = new Host("张三");
        HostProxy hostProxy = new HostProxy(host);
        hostProxy.sellHouse();
    }
}

结果：

我是中介
张三卖房得50w
您需要支付我一些费用

证明sellhouse()方法确实是由代理类执行的。

小结：

代理模式最主要的就是有一个公共接口（Person），一个具体的类（Host），一个代理类（HostProxy）,代理类持有具体类的实例，代为执行具体类实例方法。上面说到，代理模式就是在访问实际对象时引入一定程度的间接性，因为这种间接性，可以附加多种用途。这里的间接性就是指不直接调用实际对象的方法，那么我们在代理过程中就可以加上一些其他用途。
