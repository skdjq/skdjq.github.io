Bean的自动装配

byName：

需要保证所有的bean的id唯一，并且这个bean需要和自动注入的属性的set方法的值一致！

byType：

需要保证所有的bean的class唯一，并且这个bean需要和自动注入的属性类型一致！

实体类Person：

public class Person {
    private String name;
    private Cat cat;
    private Dog dog;
    //get、set、toString方法省略...
}

beans.xml：

byName：

<bean id="cat" class="com.skdjq.pojo.Cat"/>
<bean id="dog" class="com.skdjq.pojo.Dog"/>

<bean id="person" class="com.skdjq.pojo.Person" autowire="byName">
    <property name="name" value="skdjq"/>
</bean>

byType：

<bean id="cat" class="com.skdjq.pojo.Cat"/>
<bean id="dog" class="com.skdjq.pojo.Dog"/>

<bean id="person" class="com.skdjq.pojo.Person" autowire="byType">
    <property name="name" value="skdjq"/>
</bean>

注解实现自动装配

开启注解支持：

<context:annotation-config/>

@AutoWired

使用注解：

public class Person {
    private String name;
    @Autowired
    private Cat cat;
    @Autowired
    private Dog dog;
    //get、set、toString方法省略...
}

beans.xml文件：

<context:annotation-config/>

<bean id="cat" class="com.skdjq.pojo.Cat"/>
<bean id="dog" class="com.skdjq.pojo.Dog"/>
<bean id="person" class="com.skdjq.pojo.Person"/>

是不是显得清爽了很多？

如果显式的定义了@AutoWired(required = false)，则说明这个对象可以为null，否则不能为空。

@Resource

public class Person {
    private String name;
    @Resource
    private Cat cat;
    @Resource(name = "dog2")
    private Dog dog;
    //get、set、toString方法省略...
}

小结：

@Resource和@AutoWired的区别：

  • 都是用来自动装配的，都可以放在属性字段上
  • @AutoWired通过byType的方式实现
  • @Resource默认通过byName的方式实现，如果找不到名字，则通过byType实现。

使用注解开发

配置beans.xml：

<context:annotation-config/>
<!--指定要扫描的包，这个包下的注解就会生效-->
<context:component-scan base-package="com.skdjq.pojo"/>

在被扫描的包下的实体类加上：

@Component
public class Person {
    private String name;
    private Cat cat;
    private Dog dog;
}

这样我们的beans.xml可以省略这些配置了：

<bean id="cat" class="com.skdjq.pojo.Cat"/>
<bean id="dog" class="com.skdjq.pojo.Dog"/>
<bean id="person" class="com.skdjq.pojo.Person"/>

现在的beans.xml：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">

    <context:annotation-config/>
    <!--指定要扫描的包，这个包下的注解就会生效-->
    <context:component-scan base-package="com.skdjq.pojo"/>

</beans>

是不是清爽了很多？

那么如何注入属性呢？只需要使用@Value就行了：

@Component
public class Person {
    @Value("skdjq")
    private String name;
    private Cat cat;
    private Dog dog;
}
