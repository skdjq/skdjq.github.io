MySQL插入中文数据报错问题的处理
发表于 2020-06-08| 更新于 2020-06-26|数据库
字数总计:293|阅读时长: 1 分钟
|阅读量:

在MySQL插入中文数据信息常会出现报错提示：

ERROR 1366 (HY000): Incorrect string value: '\xE6\x96\xB0\xE9\x97\xBB...' for column 'title' at row1

下面是处理的方法：

我们先用语句看一下MySQL的编码方式

show variables like 'character%';

在控制台我们可以看到如下界面：

可以明显的看到database、server的编码为latinal，下面就通过修改MySQL目录下的 my.ini 来处理这个问题。

找到[mysql]，修改成如下配置：

[mysql]
default-character-set=utf8

找到找到[mysqld]，修改成如下配置：

[mysqld]
character-set-server=utf8

当修改好之后，我们重启MySQL，再次执行

show variables like 'character%';

发现

可以看到已经修改成功，我们在回到刚刚的表格重新插入中文数据看看能不能插入。

很不幸，还是报错了，还是和刚才一样的错。

因为，这个数据库是我们在没有修改编码方式的时候建立的，所以它的编码方式没有被改变，这时候我们就要重新新建一个数据库和表，这样重新创建的数据库和表就可以插入中文信息了。
