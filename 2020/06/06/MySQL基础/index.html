MySQL

MySQL数据的备份与恢复

备份的是数据库内容，并不是备份数据库！

数据库导出SQL脚本：

mysqldump -u用户名 -p密码 数据库名>生成的脚本文件的路径

例如：mysqldump -uroot -p123456 mydb1>C:\mydb1.sql

导入数据库：

第一种方法：

必须先创建数据库，然后退出。然后执行

mysql -u用户名 -p密码 要导入的数据库名<脚本文件的路径

例如：mysql -uroot -p123456 mdb2<C:\mydb1.sql

第二种方法：

登录MySQL，切换到要导入的数据库，source加SQL脚本路径

例如：source C:\mydb1.sql;

MySQL的基本命令

启动和关闭 MySQL 服务器

  • 启动：net start mysql

  • 关闭：net stop mysql

mysqld.exe是 MySQL 服务器程序。

客户端登入退出 MySQL：

  • 登录：mysql -u用户名 -p密码
    • 例如：mysql -uroot -p123456
  • 退出：quit 或 exit

mysql.exe是客户端程序。

SQL语句分类

DDL：数据库或表的结构操作

  • 创建、删除、修改：库和表的结构！

DML：对表的记录进行更新（增、删、改）

DQL：对表的记录进行查询

DCL:对用户的创建及授权

DDL

注意：所有MySQL语句后一定要以一个;结尾！

查看所有数据库：show databases;

查看所有表：show tables;

切换数据库：use 数据库名; 如use mydb1;表示切换到mydb1数据库

创建数据库：create database (if not exists) 数据库名;

（最好加上if not exists，可以避免由于已存在该数据库而报错）

删除数据库：drop database (if exists) 数据库名;

数据类型

MySQL中数据类型主要应用在列上。

int：整型

double：浮点型，例如double(5,2)表示最多5位，其中必须有2位小数，即最大值为999.99；

decimal：泛型型，在表单钱方面使用该类型，因为不会出现精度缺失问题；

char：固定长度字符串类型； 如：char(50) 必须是括号！

varchar：可变长度字符串类型； 如：varchar(50) 必须是括号！

text：字符串类型；

blob：字节类型；

date：日期类型，格式为：yyyy-MM-dd；

time：时间类型，格式为：hh:mm:ss

timestamp：时间戳类型；

在数据库中所有的字符串类型，必须使用单引号，不能使用双引号。

操作表

创建表：

create table 表名(
    列名  列类型,
    列名  列类型,
    ...
    列名  列类型
);

例如：

create table stu(
    number        char(10),
    name    varchar(50),
    age        int,
    gender    varchar(10) 
);

  • 查看当前数据库中所有表名称：show tables;

  • 查看指定表的创建语句：show create table 表名;，查看表的创建语句

  • 查看表结构：desc 表名;，查看表结构

  • 删除表：drop table 表名;，删除表

修改表：

  • 添加列：add，给stu表添加classname列：

alter table stu add (classname varchar(100));

  • 修改列类型：modify，修改stu表的gender列类型为CHAR(2)：

alter table stu modify gender char(2);

  • 修改列名：change，修改stu表的gender列名为sex：

alter table stu change gender sex char(2);

  • 删除列：drop，删除stu表的classname列：

alter table stu drop classname;

  • 修改表名称：rename，修改stu表名称为student：

alter table stu rename to student;

DML

插入数据

语法：

insert into 表名(列名1,列名2, …) values(值1, 值2)

insert into xxx(
    number,age,sex
)values(
    '1111',18,'man'
);

注意：插入的列名必须是表结构中已有的！且所有字符串必须使用单引号！

如：

mysql> desc stu1;
+--------+-------------+------+-----+---------+-------+
| Field  | Type        | Null | Key | Default | Extra |
+--------+-------------+------+-----+---------+-------+
| name   | varchar(50) | YES  |     | NULL    |       |
| age    | int(11)     | YES  |     | NULL    |       |
| gender | varchar(50) | YES  |     | NULL    |       |
| ID     | char(16)    | YES  |     | NULL    |       |
+--------+-------------+------+-----+---------+-------+

如果此时插入number，原表结构没有number这一列，那么就会出错：

mysql> insert into stu1(
    -> number,age,gender,ID)
    -> values('aaaa1111',18,'bbbb','123456');
---------------------------------------------------------
ERROR 1054 (42S22): Unknown column 'number' in 'field list'

正确的插入方法应该是：

mysql> insert into stu1(
    -> name,age,gender,ID)
    -> values('aaaa1111',18,'bbbb','123456');
---------------------------------------------------------
Query OK, 1 row affected (0.00 sec)

检查表中的列：

mysql> select * from stu1;
+----------+------+--------+--------+
| name     | age  | gender | ID     |
+----------+------+--------+--------+
| aaaa1111 |   18 | bbbb   | 123456 |
+----------+------+--------+--------+

修改数据

语法：

update 表名 set 列名1=值1,…,列名n=值n (where 条件)

条件(条件可选的)：

条件必须是一个boolean类型的值或表达式.
运算符：

  • =、!=、<>、>、<、>=、<=、

  • between…and、in(…)、is null、not、or、and

例如：

mysql> select * from stu1;
+----------+------+--------+--------+
| name     | age  | gender | ID     |
+----------+------+--------+--------+
| aaaa1111 |   18 | bbbb   | 123456 |
| zhangsan |   28 | nan    | 1111   |
| lisi     |   28 | nan    | 2222   |
+----------+------+--------+--------+
3 rows in set (0.00 sec)

我们想让name为zhangsan或者ID为2222的人的gender变为nv：

mysql> update stu1 set gender='nv'
    -> where name='zhangsan' or ID='2222';

结果为：

mysql> select * from stu1;
+----------+------+--------+--------+
| name     | age  | gender | ID     |
+----------+------+--------+--------+
| aaaa1111 |   18 | bbbb   | 123456 |
| zhangsan |   28 | nv     | 1111   |
| lisi     |   28 | nv     | 2222   |
+----------+------+--------+--------+
3 rows in set (0.00 sec)

删除数据

语法：

delete from 表名 (where 条件) 最好加上条件

DCL（了解就行）

一个项目创建一个用户！一个项目对应的数据库只有一个！

这个用户只能对这个数据库有权限，其他数据库你就操作不了了

  1. 创建用户

    • create user 用户名@IP地址 identified by ‘密码’;

      用户只能在指定的IP地址上登录

    • create user 用户名@’%’ identified by ‘密码’;

      用户可以在任意IP地址上登录

  2. 给用户授权

    • grant 权限1, … , 权限n on 数据库.* to 用户名@IP地址

      权限、用户、数据库
      给用户分派在指定的数据库上的指定的权限
      例如；grant create,alter,drop,insert,update,delete,select on mydb1.* to user1@localhost;

      • 给user1用户分派在mydb1数据库上的create、alter、drop、insert、update、delete、select权限

    • grant all on 数据库.* to 用户名@IP地址;

      给用户分派指定数据库上的所有权限

  3. 撤销授权

    • revoke 权限1, … , 权限n on 数据库.* from 用户名@IP地址;

      撤消指定用户在指定数据库上的指定权限
      例如；revoke create,alter,drop on mydb1.* from user1@localhost;

      • 撤消user1用户在mydb1数据库上的create、alter、drop权限

  4. 查看权限

    • show grants for 用户名@IP地址

      查看指定用户的权限

  5. 删除用户

    • drop user 用户名@IP地址
