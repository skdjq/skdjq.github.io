集合

为什么会出现集合——为了满足变化的需求。

数组和集合的区别

  • 长度区别

    ​ 数组的长度固定

    ​ 集合的长度可变

  • 内容不同

    ​ 数组存储的是同一种类型的元素

    ​ 集合存储的可以是不同类型的元素

  • 元素的数据类型问题

    ​ 数组既可以存储基本数据类型，也可以存储引用数据类型

    ​ 集合只能存储引用类型

集合返回长度的方法是 size() 而不是 length() ！

数组有 length 属性，但没有 length 方法！

如： arr.length 返回的是该数组的长度。

字符串有 length() 方法！

如：str.length() 返回的是该字符串的长度。

集合继承关系

Collection接口

Collection 集合的创建不能是：

Collection c = new Collection();        //error,因为接口不能实例化。

可以这样创建：

//方法一：
Collection<元素类型> 变量名 = new ArrayList<元素类型>();   ==>只能存储<>中指定的元素类型，比较常用的方式
//方法2：
Collection 变量名 = new ArrayList(); ==>集合的元素默认为Object类型，可以存储任何类型的元素

Collection 接口的基本方法

boolean add(Object obj)：添加一个元素 void clear()：移除collection中所有元素（集合还在） boolean contains(Object obj)：判断对象是否存在集合中 boolean remove(Object obj)：移除集合中指定的元素（删除第一个遇到的元素） int size()：返回集合中元素的个数 Object[] toArray()：把集合中的元素转成数组中的元素（集合转成数组）

public class demo1 {
    public static void main(String[] args) {
        Collection c = new ArrayList();
        c.add("hello");
        c.add("world");
        c.add("!");
        Object[] obj = c.toArray();
        for (int i = 0; i < obj.length; ++i) {
            String s = (String) obj[i];     //向下转型
            System.out.println(obj[i] + "----" + s.length());
        }
    }
}

Iterator 迭代器

Iterator iterator() : 迭代器，集合的专用遍历方式。

  • boolean hasNext()：判断集合中是否还有元素可以迭代
  • next()：获取出可以迭代的下一个元素
  • void remove()：从迭代器指向的集合中移除迭代器返回的最后一个元素

逻辑实例：

public class demo2 {
    public static void main(String[] args) {
        Collection c = new ArrayList();
        c.add("hello");
        c.add("world");
        c.add("!");
        Iterator it = c.iterator();
        while(it.hasNext()) {
            System.out.println(it.next());
        }
    }
}
