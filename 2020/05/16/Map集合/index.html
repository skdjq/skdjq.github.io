Map集合

Map集合

Map集合的特点

  • 每个元素成对存在，由键和值两部分组成，通过键可以找到对应的值

  • 键（key值）不可重复，值（value）可以重复，一个value值可以和

    很多key值形成对应关系，每个建最多只能映射到一个值

  • 两个 Map 接口的重要实现类：HashMap 类、LinkedHashMap 类

以学生学号为例，学号就是键，姓名就是值：

  • 学号1 姓名1
  • 学号2 姓名2
  • 学号1（不行） 姓名3
  • 学号3 姓名2（可以）

Map集合和Collection集合的区别：

  • Map集合存储元素是成对出现的，Map集合的键是唯一的，值是可重复的。
  • Collection集合存储元素是单独出现的，Collection的儿子Set是唯一的，List是可重复的。

注意：

  • Map集合的数据结构只针对键有效，和值无关。
  • Collection集合的数据结构是针对元素有效。

Map接口中的常用方法

  • get(Object key)：获取指定键(key)所对应的值(value)
  • put(K key，V value)：将指定的键与值对应起来，并添加到集合中
    若指定的键(key)在集合中没有，则没有这个键对应的值，返回null，并把指定的键值添加到集合中
    若指定的键(key)在集合中存在，则返回值为集合中键对应的值（该值为替换前的值），并把指定键所对应的值，替换成指定的新值
  • remove(Object key)：根据指定的键(key)删除元素，返回被删除元素的值(value)

逻辑实例：

public static void main(String[] args)
{
    //创建Map对象
    Map<String,String> M = new HashMap<>();
    //添加元素
    M.put("汤姆","Tom");
    M.put("提姆","Tim");
    M.put("瑞克","Rik");
    System.out.println(M);

    //根据指定的key获取对应的value
    System.out.println(M.get("汤姆"));

    //根据key删除元素,会返回key对应的value值
    String val = M.remove("提姆");
    System.out.println(val);
    System.out.println(M);

    //当给Map中添加元素，会返回key对应的原来的value值,若key没有对应的值，返回null
    System.out.println(M.put("汤姆","TTT"));  //Tom
    System.out.println(M);
}

遍历

思路：

  • 获取所有的键
  • 遍历键的集合，获取得到每一个键
  • 根据键去找值

Map集合中提供了两种取出方式：

<1>. 返回值类型Set<k> 方法是： keySet() :返回此映射中包含的键的 Set 视图
将 map 中所有的键存入到 Set 集合，因为 set 具备迭代器，所有迭代方式取出所有的键
再根据 get() 方法 ，获取每一个键对应的值

<2>. 返回值类型：Set<Map.Entry<K,V>>方法是：entrySet()
取出的是关系，关系中包含 key 和 value ,其中 Map.Entry<k,V> 来表示这种数据类型
即：将 map 集合中的映射关系存入到set集合中，这个关系的数据类型为: Map.Entry

注意：Map集合中是没有迭代器的 ,Map集合取出键值的原理：将Map集合转成Set集合，再通过迭代器取出 。

keySet 方法

keySet() :返回此映射中包含的键的 Set 视图

public class MapDemo1 {
    public static void main(String[] args) {
        Map<String, String> m = new HashMap<String, String>();
        m.put("张三", "18");
        m.put("张小三", "28");
        m.put("张三丰", "18");
        Set<String> set = m.keySet();                //找出所有的键
        for(String s : set) {                        //遍历所有的键
            String ss = m.get(s);                    //根据键去找值
            System.out.println(s + "----" + ss);
        }
    }
}

entrySet 方法

Entry 对象特点：

Entry是Map接口中提供的一个静态内部嵌套接口
相关方法

  • getKey()方法：获取Entry对象中的键

  • getValue()方法：获取Entry对象中的值

  • entrySet()方法：用于返回Map集合中所有的键值对(Entry)对象，以Set集合形式返回

Map 集合遍历方式 entrySet 方法：

  • 调用 Map 集合方法 entrySet() 将集合中的映射关系对象 Entrty 存储到Set集合：Set<Entry <K,V> >
  • 迭代 Set 集合
  • 获取出的 Set 集合的元素，是映射关系对象
  • 通过映射关系对象方法 getKey, getValue 获取键值对
public class MapDemo1 {
    public static void main(String[] args) {
        Map<String, String> m = new HashMap<String, String>();
        m.put("张三", "18");
        m.put("张小三", "28");
        m.put("张三丰", "18");
        Set<Map.Entry<String, String>> set = m.entrySet();
        for(Map.Entry<String, String> k : set) {
            String key = k.getKey();
            String value = k.getValue();
            System.out.println(key + "----" + value);
        }
    }
}

keySet方法和entrySet方法的对比

常见问题：

Entry是接口 ，但是为什么不定义定义在外面呢？

原因：

  • Entry代表的是映射关系，先有Map集合，才有的映射关系，所以它是Map集合内部的事物，因此将Entry定义为
    在Map的内部集合，可以直接访问Map集合中的元素。
  • 在API文档中有 public static interface Map.Entry<K,V> 其中被static静态修饰，只有接口在
    成员位置上才能加静态修饰符 ,说明其是内部接口。

练习

利用Map集合获取一个字符串的每个字符出现的次数，并以字符串形式输出。

如：aabbccdd 输出结果为：a：2次 b：2次 c：2次 d：2次

思路：

  • 将字符串转化为数组
  • 创建一个Map集合，此时里面为空
  • 遍历数组，将遍历得到的字符当作Map中的key
    • 如果Map中没有相同的key，则将value值初始化为1
    • 如果Map中已有相同的key，则将value值加一
  • 遍历Map集合，打印输出结果

逻辑实例：

public class MapDemo2 {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);
        String s = scan.next();
        System.out.println(s);
        //字符串转化为数组
        char[] arr = s.toCharArray();
        //创建一个Map集合
        Map<Character, Integer> map = new HashMap<Character, Integer>();
        //判断Map集合中的key，如果没有相同的key， value为1；如果有相同的key， value加1
        for (int i = 0; i < arr.length; ++i) {
            if (!map.containsKey(arr[i])) {
                map.put(arr[i], 1);
            } else {
                map.put(arr[i], map.get(arr[i]) + 1);
            }
        }
        //遍历Map集合
        Set<Character> set = map.keySet();
        for (Character k : set) {
            Integer value = map.get(k);
            System.out.println(k + ":" + value + "次");
        }
    }
}

HashMap 嵌套 HashMap

当我们有类似这样一个结构时，就需要使用HashMap的嵌套了。

如：

AHUT

  • 电气
    • 电气工程及其自动化 200
    • 电子信息 180
  • 数理
    • 光电信息科学与工程 80
    • 光源与照明 80

这就是一个典型的HashMap的嵌套。

逻辑实例：

public class MapDemo3 {
    public static void main(String[] args) {
        //创建一个大Map
        Map<String, HashMap<String, Integer>> BigMap = new HashMap<String, HashMap<String, Integer>>();
        //创建电气Map
        HashMap<String, Integer> Dmap = new HashMap<String, Integer>();
        Dmap.put("电气工程及其自动化", 200);
        Dmap.put("电子信息工程", 180);
        BigMap.put("电气", Dmap);
        //创建数理Map
        HashMap<String, Integer> Smap = new HashMap<String, Integer>();
        Smap.put("光电信息科学与工程", 80);
        Smap.put("光源与照明", 80);
        BigMap.put("数理", Smap);
        //遍历嵌套Map
        Set<String> BigSet = BigMap.keySet();
        for (String k1 : BigSet) {
            System.out.println(k1);
            HashMap<String, Integer> value1 = BigMap.get(k1);
            Set<String> SmallSet = value1.keySet();
            for (String k2 : SmallSet) {
                Integer value2 = value1.get(k2);
                System.out.println(k2 + "----" + value2);
            }
        }
    }
}

面试题

HashMap 和 Hashtable 的区别

Hashtable：

  • 线程安全，效率低
  • 不允许 NULL 键和 NULL 值

HashMap：

  • 线程不安全，效率高
  • 允许 NULL 键和 NULL 值

List, Set, Map 等接口是否都继承自 Map 接口？

  • List，Set不是继承自Map接口，他们继承自Collection接口
  • Map接口本身就是一个顶层接口
